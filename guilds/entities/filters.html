Command Filters

Command filters are consistent across every command that is part of the guilds plugin's engine. Exceptions are the /guilds command itself. Whenever the help screen pops up, or whenever the suggestions tab previews the command, it will show all of the filters in a <filter> syntax.

Numerical <#>

[1.0.0] This filter represents whole numbers. Any number positive or negative can be typed in. This does not autocomplete in a command.

Wildcard <name>

[1.0.0] This filter takes in any input so long as there are no spaces. This is a generic prompt that is usually used for searching or setting names. This does not autocomplete in a command.

Input <player>

[1.0.0] This filter represents every player that has joined the server. Most player prompts use this filter unless for specific reasons. This will autocomplete by finding every single player ever (if under 40 players).

Input <nplayer>

[1.0.0] This filter represents every player that is part of your nation. This includes people in states and towns that are also in the nation. You will need to be part of a nation to use any commands that have this filter.

Input <splayer>

[1.0.0] This filter represents every player that is part of your state. This includes people in other towns. You will need to be part of a state to use any commands that have this filter.

Input <townsman>

[1.0.0] This filter represents every player that is part of your town. This includes people in states and towns that are also in the nation. You will need to be part of a town to use any commands that have this filter.

Trait <color>

[1.0.0] This filter involves colors. You can use primary colors, or HTML colors. To get a HTML color, refer either to your nearest image editor, or use this: Some servers may make /n color a paid feature.

Trait <title>

[1.0.0] This filter involves ranks, or specifically rank titles. Titles are uniform across nations, states, and towns. The autofiller is good at presenting each title if you do not know all of them.

Holding <nation>

[1.0.0] This filter involves all nations that are listed. Any type of nation will work. However, Safezone is not considered a listed nation.

Holding <state>

[1.0.0] This filter involves all states, independent or under a nation. If you are looking up California, you may use California, or to further clarify, you may use California,USA. This does not autofill in a command, take note that you must not use a space. California, USA will not work.

Holding <Town>

[1.0.0] This filter involves all towns on the server. If you are looking up LosAngeles, you may use LosAngeles, or if there's multiple towns of that name, you may use LosAngeles,USA or LosAngeles,California,USA. This does not autofill in a command, take note that you must not use a space. Los Angeles, California, USA will not work.

Guild <signature>

[1.0.0] This filter involves signed war declarations. You will need to use the signature's UID. Currently the war system is not in use.
