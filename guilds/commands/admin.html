/ga safezone

Take ownership of the built-in safezone. This is best to use if the server owner prefers a guilds-native spawn. It is advised to use, or combine with a proper spawn plugin

/ga dynmap

Display claims on dynmap. This is normally done automatically, but can be forcefully done, such as for debug purposes.

/ga claim (nation)

If a nation needs to claim a chunk by force, this command is here. This can only be done by server admins and owners.

/ga playerdata

If guilds was installed at a later point, this will take all of the players that joined the server, and properly register them into a guilds-native instance.

/ga nation (nation) (player)

Create a nation on a player's behalf. This normally does not need to be done as a player will be able to make one themselves.

/ga assign (nation) (player)

If a player is unable to join a nation, you are able to use this command to put them in the nation by force.

/ga king (nation) (player)

If a nation is leaderless, a nation's leader went inactive, or some other circumstance is at play, you are able to transfer leadership over a nation to the selected player.

/ga searchtown (string)

Debug state search engine. Currently does not do much aside from generate a list of towns that contains the letters.

/ga searchstate (string)

Debug state search engine. Currently does not do much aside from generate a list of states that contains the letters.

/ga searchnation (string)

Debug state search engine. Currently does not do much aside from generate a list of nations that contains the letters.
